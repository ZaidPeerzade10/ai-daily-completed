--- High-Level Deployment and Monitoring Considerations ---
This trained model can be deployed as an API endpoint to score new customers periodically.
When predicting for a new customer (e.g., CUST_NEW), the same feature engineering pipeline (steps 3-4) would be applied:
  1. Define a current 'global_analysis_date' (e.g., today's date).
  2. Calculate 'feature_cutoff_date' (today - 60 days).
  3. Extract static features and calculate 'account_age_at_feature_cutoff'.
  4. Aggregate usage events for CUST_NEW within the last 60 days leading up to 'feature_cutoff_date'.
  5. Use the preprocessor and trained classifier to get a churn probability for CUST_NEW.
Continuous monitoring of model performance (e.g., ROC AUC, precision/recall for actual churn vs. predicted churn) is crucial.
Data drift (changes in feature distributions) and concept drift (changes in churn behavior patterns) necessitate periodic retraining of the model with fresh data.
This script provides a solid foundation for a comprehensive churn prediction system.
